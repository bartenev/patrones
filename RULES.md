# Правила создания дриллов (Patrones)

Спека для генерации `unidad-NN.json` из разворота учебника и для работы движка.
Держать рядом с движком и юнитами.

---

## 1. Структура и оформление JSON

Один файл на юнит, имя `unidad-NN.json`.

```json
{
  "unit": "Unidad N — Название темы",
  "blocks": [
    {
      "title": "Название блока",
      "mode": "transform|vocab|cloze|phrase",
      "cards": [
        { "front": "...", "back": "...", "note": "...", "translation": "..." }
      ]
    }
  ]
}
```

Карточка — объект с фиксированным порядком ключей: `front`, `back`, `note`, `translation`.
- `front`, `back` — обязательны.
- `note` — необязательное пояснение правила/ловушки.
- `translation` — необязательный перевод.

Оформление:
- **одна карточка — одна строка**.
- пустые необязательные поля **не пишутся** (нет пустых `note`/`translation`).
- порядок ключей всегда один: front → back → note → translation (даже если что-то опущено).
- отступы — 2 пробела для обёртки; НЕ разбивать объект карточки на несколько строк.

---

## 2. Нарезка на блоки

Основные принципы:

- **Один блок = одна операция** (одно мысленное действие), а не «одна тема».
- **Правило и исключение не смешиваются никогда**, даже если поверхностно похожи.
  (`estudiante` = правило -ista/-ante; `joven` = исключение → разные блоки.)
- **Разные окончания можно держать вместе, если операция одна.**
  (-ista + -ante → один блок; -ción/-sión/-dad/-tad → один блок.)
- **Более специфичный признак важнее общего.**
  (`león → leones` — не в «согласная +es», а в «уходит тильда».)
- **Зеркальные ловушки разводятся.**
  («-o, но женский» и «-a, но мужской» — два блока.)
- **Никаких блоков-свалок.** «Слова из упражнения» с перемешанными родами/операциями запрещены — раскидать по блокам-правилам.
- **Каждая карточка репрезентативна для своего блока.** Любая случайно взятая карта должна быть типичным примером того, что обещает заголовок. Если карта нетипична — она в неверном блоке (см. §6, проверка «заголовок = содержимое»).
- **Списочные блоки допустимы** (лексика, «особые формы», «разные слова») — там объединяющий признак не операция, а категория. Это законно, если категория честная и однородная.

> Списочные блоки собирают непредсказуемые формы (actriz, reina, alcaldesa; médico/médica) и лексические пары (marido/mujer). Их **не** выносим из дрилла — они дриллуются наравне с правилами.

## 3. Типы карточек (mode) и translation

| mode | операция | стороны | вопрос → ответ |
|------|----------|---------|----------------|
| **transform** | преобразовать форму | ES → ES | front → back |
| **vocab** | вспомнить слово по смыслу | RU → ES | front(RU) → back(ES) |
| **cloze** | заполнить пропуск | фраза с `___` → заполнение | front → back |
| **phrase** | произвести реплику | ES → ES | front → back |

**transform** — `el niño → la niña`; `libro → libros`.

**vocab** — `город → una ciudad`. Единственный тип RU→ES. Существительные ВСЕГДА с артиклем.

**cloze**:
- `back` = **только заполнение**, не вся фраза (`una`, `un`, `∅`).
- «ничего не ставится» → символ **`∅`** (не пробел, не пустая строка).
- `note` = сработавшее правило.
- `translation` = **перевод всей фразы** по-русски (перевод одного пропуска бессмыслен).

**phrase** — `¿Qué es Sevilla? → Es una ciudad.` Редкий тип: применяется, когда ответ — цельная порождаемая реплика, а не одно слово в пропуск. Если проверяется выбор артикля/формы внутри готовой фразы — это **cloze**, а не phrase.

### Правило translation (общее для всех mode)

- **translation привязан к смыслу карточки, а не к конкретному полю.**
- Показывается на **стороне-вопросе** — но только если вопрос по-испански И перевод не совпадает с ответом.
- Если вопрос уже русский (vocab) или ответ сам является переводом — translation не показывается (иначе спойлер). Для `phrase` глосса показывается на стороне ОТВЕТА по той же причине.
- Существительные в translation и в back — **всегда с артиклем** (`la mano`, не `mano`).

## 4. Что класть в дек: покрытие и отбор

Дек генерируется под конкретный юнит и должен **покрывать существительные и формы, встречающиеся в теории и упражнениях этого юнита**. Но покрытие — не «затащить все слова подряд».

**Приоритет — инструктивные единицы.** Обязательно включаются:
- ловушки (род/форма вопреки окончанию: `la mano`, `el día`, `un mapa`);
- исключения из правила (`el joven → la joven`; `el piloto → la piloto`);
- слова-маркеры окончаний, ради которых существует блок (`universidad` для -dad, `postal` для согласной +es);
- слова из упражнений, иллюстрирующие ровно ту механику, что даёт юнит.

**Regular-filler включается дозированно.** Чисто регулярные примеры (`botella`, `silla`, `vaso` для -a→+s) нужны лишь в объёме, достаточном, чтобы паттерн закрепился. Как только паттерн в блоке представлен плотно, добавлять новые однотипные слова из упражнения не требуется.

**Критерий отбора одной фразой:** *«эта карточка учит новому решению — или просто повторяет уже плотно отработанное?»* Первое включаем всегда, второе — по мере надобности.

**Оценка покрытия.** Полезно после сборки прикинуть: какие слова из теории и упражнений юнита не попали в дек, и среди них — есть ли инструктивные. Инструктивные пробелы закрываются; filler-пробелы допустимы и отмечаются как осознанный пропуск.

---

## 5. Педагогические принципы (наполнение)

- **Новое слово сперва как лексика, потом грамматический дрилл.** Незнакомое слово в грамматическом дрилле = двойная нагрузка. Отсюда поле `translation`.
- **Порядок работы:** блоки-правила по отдельности (постановка) → «перемешать всё» (экзамен). Смешивать до постановки — вредно. Экзаменационную роль выполняет режим движка «перемешать всё», отдельного микс-блока в JSON не заводим.
- **note — микро-подсказка**, а не пересказ теории (`-ma, но м.р.`; `+ прилаг. → una`). Полное правило — в учебнике.
- **Дубли между блоками.** Одно слово попадает в два блока, только если оно иллюстрирует **две разные операции** (напр. явный контраст правил). Если операция по сути одна — слово живёт в одном блоке, выбранном по более специфичному признаку. Не плодить одно и то же слово с несогласованными пометками.

---

## 6. Чек-лист самопроверки (перед выдачей JSON)

1. **Заголовок = содержимое.** По каждому блоку: подходит ли КАЖДАЯ карточка под название? Классические сбои: слово на согласную в блоке «-o/-e → -a»; слово на -e в блоке «согласная». Не подходит → перенести в верный блок (возможно, придётся создать новый).
2. **Правило ≠ исключение** внутри одного блока. Исключения вынесены отдельно.
3. **Зеркальные ловушки** («-o но ж.р.» и «-a но м.р.») в разных блоках.
4. **Покрытие.** Все инструктивные слова юнита (ловушки, исключения, маркеры) в деке. Filler-пробелы осознаны.
5. **Дубли.** Ни одно слово не дублируется между блоками без двух разных операций.
6. **cloze:** `back` — только заполнение; «пусто» = `∅`; `translation` — вся фраза.
7. **Артикли:** все существительные в `vocab`-back и в `translation` — с артиклем.
8. **Оформление:** карта в строку; порядок ключей front→back→note→translation; пустые поля опущены.

---

## 7. Промпт для генерации нового юнита

> По приложенному развороту учебника Gramática de uso del español собери `unidad-NN.json` строго по RULES.md:
>
> **Нарезка.** Раздели на блоки по ОПЕРАЦИИ (один блок = одно действие). Правило и исключение — врозь; зеркальные ловушки — врозь; свалок нет; каждая карта репрезентативна для своего блока.
>
> **Типы.** Выбери mode для каждого блока (transform / vocab / cloze / phrase). cloze: `back` = только заполнение (артикль или `∅`), `translation` = вся фраза. phrase — только когда ответ это цельная реплика, а не подстановка.
>
> **Наполнение.** Покрой инструктивные единицы юнита (ловушки, исключения, слова-маркеры окончаний) и слова из упражнений, иллюстрирующие механику юнита. Регулярный filler — дозированно, лишь пока паттерн не закреплён. Существительные — с артиклем; новые слова снабди `translation`; note — микро-подсказка.
>
> **Дубли.** Слово в двух блоках — только при двух разных операциях; иначе один блок по специфичному признаку.
>
> **Оформление.** Карточка в строку; ключи front→back→note→translation; пустые поля опускать.
>
> **Перед выдачей** прогони чек-лист §6 (особенно «заголовок = содержимое» и покрытие инструктивных слов). Верни валидный JSON и коротко перечисли, какие слова юнита осознанно не вошли и почему.
