# Patrones

Паттерн-тренажёр испанской грамматики. Vue 3 + Vite, карточки из JSON-файлов в `decks/`, прогресс и банк ошибок — в **IndexedDB** в браузере.

Спека формата колод: [`RULES.md`](./RULES.md).

---

## Быстрый старт

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # production-сборка в dist/
npm run preview    # превью сборки
npm test           # unit-тесты (vitest)
npm run test:coverage
```

Для доступа с телефона в локальной сети скопируй `.env.example` → `.env` и укажи IP машины:

```env
DEV_HOST=192.168.0.116
DEV_PORT=5173
```

## Настройка прогона

Панель разбита на две вкладки: **Контент** и **Режим**.

### Контент

- **Тип карточек** — фильтр по `mode` блока: `transform`, `vocab`, `cloze`, `phrase`. Можно включить несколько или все.
- **Юниты** — список файлов из `decks/*.json`. Клик по строке включает/выключает юнит.
- Кнопки **все** / **снять** — выделить или сбросить все юниты.
- **блоки** — попап выбора отдельных блоков внутри юнита (вкл/выкл блоки, «все» / «снять» внутри попапа). Закрытие: кнопка «Готово», крестик, клик по оверлею, `Esc`. При снятии юнита попап закрывается автоматически.
- Счётчик в кнопке старта показывает число пар с учётом выбранных юнитов, блоков и фильтра типов.

### Режим

- **Порядок карточек** — 7 режимов (см. ниже).
- **Сессия** — направление, озвучка, повтор ошибок, таймер.

---

## Режимы порядка (7)

| # | Режим | Поведение |
|---|-------|-----------|
| 1 | **Всё по порядку** | Юниты по очереди → блоки прямо → карточки прямо |
| 2 | **Карточки вперемешку** | Юниты по очереди → блоки прямо → карточки внутри блока случайно |
| 3 | **Блоки и карточки вперемешку** | Юниты по очереди → блоки случайно → карточки внутри блока случайно |
| 4 | **Полный хаос** | Все карточки выбранных юнитов в один случайный поток |
| 5 | **Только ошибки** | До **10** карточек из банка ошибок (IndexedDB). Счётчик `(N)` в селекте. Ошибся — снова в очередь; «Знал» — убирается из банка. Направление берётся из сохранённой ошибки |
| 6 | **Ревью** | По **одной** случайной карточке из каждого активного блока выбранных юнитов |
| 7 | **Слабые** | До **10** карточек с высокой долей ошибок по прогрессу (IndexedDB). Счётчик `(N)` в селекте. Учитываются выбранные юниты, фильтр типов и текущее направление |

Режимы **5** и **7** отключают кнопку старта, если банк/слабые пусты.

## Параметры сессии

| Параметр | Описание |
|----------|----------|
| **Направление** | `front → back` или `back → front` |
| **Озвучивать ответ** | При показе ответа — `speechSynthesis` (es-ES, предпочтение голоса `es-ES`) |
| **Повторять ошибки** | При «Споткнулся» карточка возвращается в хвост очереди (в обычных режимах; в режиме ошибок — всегда) |
| **Таймер** | `вручную` или **1–5 сек** на вопрос + фиксированная **1 сек** на ответ после раскрытия |

### Таймерный режим

- Вопрос открывается автоматически по истечении таймера.
- После раскрытия — 1 с на просмотр ответа, затем переход к следующей карточке **без оценки** (прогресс и банк ошибок при авто-переходе **не** обновляются).
- **Клик по карточке** — пауза / продолжение таймера.
- Полоска-таймер на карточке.
- Кнопки «Споткнулся» / «Знал» скрыты — только авто-переход.

---

## Дрилл (ручной режим)

- **Показать ответ** — раскрывает `back` (или обратную сторону при `rev`).
- **Споткнулся** / **Знал** — оценка после раскрытия.
- На стороне вопроса может показываться `translation` (по правилам типа карточки).
- После раскрытия — `note`, кнопка **🔊 произнести**.
- Полоса прогресса, счётчики «осталось» и «споткнулся».
- Подпись юнита и название блока в `secbar`.

### Горячие клавиши (только в дрилле, без таймера)

| Клавиша | Действие |
|---------|----------|
| `Пробел` / `Enter` | Показать ответ |
| `←` / `1` | Споткнулся |
| `→` / `2` | Знал |
| `S` | Озвучить испанский текст |
| `Esc` | Выход к настройке |

## Типы карточек (`mode`)

| mode | Стороны | Смысл |
|------|---------|-------|
| **transform** | ES → ES | Преобразование формы (`el niño → la niña`) |
| **vocab** | RU → ES | Слово по переводу (`город → una ciudad`) |
| **cloze** | фраза с `___` → заполнение | `back` — только подстановка; пусто = `∅` |
| **phrase** | ES → ES | Цельная реплика (`¿Qué es…? → Es una…`) |

Логика отображения сторон (`sideFor`) учитывает направление и тип: для `vocab` вопрос — русский `front`, ответ — испанский `back`.

## Формат колод

- Файлы: `decks/unidad-NN.json`, подхватываются через Vite `import.meta.glob` при сборке.
- Структура: `unit` + массив `blocks[]` с `title`, `mode`, `cards[]`.
- У каждой карточки обязателен уникальный **`uuid`** (v4) — без него прогресс и банк ошибок для карточки не ведутся.
- Первый юнит в списке включается по умолчанию при загрузке.
- Битые JSON попадают в сообщение об ошибке на экране настройки.

Подробные правила нарезки блоков и оформления JSON — в [`RULES.md`](./RULES.md).

## Хранение данных (IndexedDB)

База **`patrones`**, версия **2**. Данные локальные, привязаны к origin браузера. Очистка данных сайта в браузере удалит прогресс и ошибки.

### Store `lessons` — прогресс

- Ключ: `lessonId` (имя файла, напр. `unidad-01.json`).
- На карточку (`uuid`): отдельная статистика для `fwd` и `rev` — `correct`, `wrong`, `streak`, `lastSeenAt`.
- Обновляется при каждой ручной оценке «Знал» / «Споткнулся».

**Режим «Слабые»** — карточки, у которых:

- не меньше **2** попыток в выбранном направлении;
- weakness = `wrong / (attempts + 2)` ≥ **0.3**.

Сортировка по убыванию weakness, в сессию — до **10** штук.

### Store `mistakes` — банк ошибок

- Ключ: `uuid` + направление (`fwd` / `rev`).
- Запись при «Споткнулся» (с `missCount`, `lastMissedAt`, снимок карточки).
- Удаление при «Знал» в режиме **«Только ошибки»**.
- В сессию — до **10** последних по `lastMissedAt`.

Миграции со старых форматов (localStorage и т.п.) **нет**.

## Что не реализовано

- Сохранение настроек UI (тема, таймер, порядок, фильтры) между сессиями.
- Кнопка «Повторить ошибки» на экране ¡Listo! (только общий рестарт и возврат к настройке).
- Запись прогресса / ошибок при авто-переходе по таймеру.
- Экспорт / импорт прогресса и банка ошибок.
