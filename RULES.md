# Правила создания дриллов (Patrones)

Спека для генерации `unidad-NN.json` и работы движка. Держать рядом с движком и юнитами.

---

## 1. Структура и оформление JSON

Один файл на юнит, имя `unidad-NN.json`.

```json
{
  "unit": "Unidad N — Название темы",
  "blocks": [
    {
      "title": "Название блока",
      "mode": "transform|vocab|cloze|phrase",
      "cards": [
        { "front": "...", "back": "...", "note": "...", "translation": "..." }
      ]
    }
  ]
}
```

Карточка — объект с фиксированным порядком ключей: `front`, `back`, `note`, `translation`.
- `front`, `back` — обязательны.
- `note` — необязательное пояснение правила/ловушки.
- `translation` — необязательный перевод.

Оформление:
- **одна карточка — одна строка**.
- пустые необязательные поля **не пишутся** (нет пустых `note`/`translation`).
- порядок ключей всегда один: front → back → note → translation (даже если что-то опущено).
- отступы — 2 пробела для обёртки; НЕ использовать indent, разбивающий объект карточки на строки.
- флаг блока `"mix": true` — опциональный, ставится сразу после `mode`.

---

## 2. Нарезка на блоки — главные принципы

- **Один блок = одна операция** (одно мысленное действие), а не «одна тема».
- **Правило и исключение не смешиваются никогда**, даже если поверхностно похожи.
  (estudiante = правило -ista/-ante; joven = исключение → разные блоки.)
- **Разные окончания можно держать вместе, если операция одна.**
  (-ista + -ante → один блок; -ción/-sión/-dad/-tad → один блок.)
- **Более специфичный признак важнее общего.**
  (león → leones: не в «согласная +es», а в «уходит тильда».)
- **Зеркальные ловушки разводятся.**
  («-o, но женский» и «-a, но мужской» — два блока.)
- **Никаких блоков-свалок.** «Слова из упражнения» с перемешанными родами запрещены — раскидать по блокам-правилам.
- **Списочные блоки (лексика, особые формы) допустимы**, но ревью-мод проверяет их лишь точечно.
- **Однородность идёт от ревью-мода:** любая случайная карта из блока должна быть репрезентативна для блока.
- **Намеренно смешанный блок** помечается `"mix": true` (чтобы ревью-мод его пропускал). Но если в движке есть «перемешать всё» — микс-блок обычно лишний: расформировать по правило-блокам, роль экзамена отдать shuffleAll.

---

## 3. Типы карточек (mode)

| mode | операция | стороны | направление |
|------|----------|---------|-------------|
| **transform** | преобразовать форму | ES → ES | front→back |
| **vocab** | вспомнить слово по смыслу | RU → ES | RU→ES |
| **cloze** | заполнить пропуск | фраза с `___` → заполнение | front→back |
| **phrase** | произвести реплику | ES → ES | front→back |

- **transform** — el niño → la niña; libro → libros.
- **vocab** — город → una ciudad. Единственный тип RU→ES. Существительные ВСЕГДА с артиклем.
- **cloze**:
  - `back` = **только заполнение**, не вся фраза (`una`, `un`, `∅`).
  - «ничего не ставится» → символ **`∅`** (не пробел, не пустая строка).
  - `translation` = **перевод всей фразы** по-русски (перевод пропуска бессмыслен).
  - `note` = сработавшее правило.
- **phrase** — ¿Qué es Sevilla? → Es una ciudad. `translation` = глосса, показывается на стороне ОТВЕТА (иначе спойлер).

Для движка практически различается только `vocab` (RU→ES) против остальных (front→back). Имена типов — семантические.

---

## 4. translation и направление

- **translation привязан к смыслу карточки, а не к полю.**
- Показывается на **стороне-вопросе**, но только если та испанская И перевод ≠ ответу.
- Если вопрос уже русский или ответ и есть перевод — не показывается (спойлер).
- Существительные — **всегда с артиклем** (не `mano`, а `la mano`).

---

## 5. Педагогические принципы (наполнение)

- **Новое слово сперва как лексика, потом грамматический дрилл.** Незнакомое слово в дрилле = двойная нагрузка. Отсюда поле `translation`.
- **Регулярное → дрилл; исключения и произвольные факты → Anki.** Весь сет в Anki не копируется — только непредсказуемая часть.
- **Порядок:** блоки-правила по отдельности (постановка) → «перемешать всё» (экзамен). Смешивать до постановки — вредно.
- **note — микро-подсказка** (`-ma, но м.р.`, `+ прилаг. → una`), не пересказ теории. Полное правило — в учебнике.

---

## 6. Промпт для генерации нового юнита

> По приложенному развороту учебника Gramática de uso del español собери `unidad-NN.json` строго по правилам RULES.md:
> — раздели на блоки по ОПЕРАЦИИ (один блок = одно действие), правило и исключение — врозь, зеркальные ловушки — врозь, свалок нет;
> — выбери mode для каждого блока (transform/vocab/cloze/phrase);
> — cloze: back = только заполнение (артикль или ∅), translation = вся фраза;
> — существительные с артиклем; новые слова снабди translation;
> — оформление: карточка в строку, ключи front→back→note→translation, пустые поля опускать;
> — верни валидный JSON.
